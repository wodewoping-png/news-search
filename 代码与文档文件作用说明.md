# 代码与文档文件作用说明

本文档用于快速理解 `news-search` 仓库下各类代码、配置、数据和文档文件的职责。

项目整体是一个多源能源/科技新闻抓取系统：每天从 RSS 和 HTML 列表页抓取前一天新闻，生成 CSV 文件，并通过 GitHub Actions 自动提交到仓库。

## 一、项目总览

### 核心目标

- 从能源、光伏、电池、储能、氢能、氨能、核电、风电和综合科技网站抓取新闻。
- 每日按 UTC 日期计算“昨日新闻”，输出 `daily-news/daily_news_YYYYMMDD.csv`。
- 默认只抓标题、URL、发布时间等结构化信息，不抓正文，以保证运行速度和稳定性。
- 通过 GitHub Actions 每天北京时间 12:00 自动运行并 push 结果。

### 主要目录

| 路径 | 作用 |
|------|------|
| `.github/workflows/` | GitHub Actions 自动化任务配置 |
| `carbon_spider/` | 新闻抓取程序源码 |
| `carbon_spider/configs/` | 站点清单和抓取入口配置 |
| `carbon_spider/spiders/` | RSS 和 HTML 抓取模块 |
| `carbon_spider/utils/` | 时间解析、日期过滤等工具函数 |
| `daily-news/` | 每日抓取结果 CSV 归档，已入库 |
| `docs/` | 代码逻辑、历史修复和已知问题说明 |

## 二、代码文件说明

### `carbon_spider/main.py`

项目主入口，负责串联完整抓取流程。

主要职责：

- 读取 `carbon_spider/configs/sites.yaml` 中的站点配置。
- 根据站点 `type` 分发到 RSS 抓取或 HTML 抓取。
- 默认用 `is_yesterday()` 只保留昨日新闻。
- 支持 `--all-dates` 调试模式，跳过日期过滤。
- 支持正文抓取，但默认建议使用 `--no-content` 跳过正文。
- 将结果写入 `carbon_spider/outputs/daily_news_YYYYMMDD.csv`。

常用命令：

```bash
cd carbon_spider
python3 main.py --no-content
python3 main.py --no-content --all-dates
```

输出字段为：

| 字段 | 含义 |
|------|------|
| `date` | 归档日期，格式 `YYYYMMDD`，表示新闻日期 |
| `site` | 站点配置名称 |
| `category` | 新闻分类 |
| `title` | 新闻标题 |
| `url` | 原文链接 |
| `publish_time` | 发布时间 |
| `content` | 正文内容，默认空 |

### `carbon_spider/spiders/rss_spider.py`

通用 RSS/Atom 抓取模块。

主要职责：

- 使用 `feedparser` 解析 RSS 或 Atom feed。
- 从条目中提取 `title`、`link`、发布时间。
- 发布时间字段按 `published`、`updated`、`created`、`published_parsed` 等顺序兜底。
- 当 feed 格式异常但仍可解析时记录 warning，不中断整个任务。
- 返回统一结构的文章字典，供 `main.py` 后续过滤和输出。

适用场景：

- 标准 RSS 站点。
- Google News RSS 聚合源，例如 `h2_view`、`perovskite_info`。

### `carbon_spider/spiders/html_spider.py`

HTML 列表页抓取模块，也是站点适配逻辑最集中的文件。

主要职责：

- 提供 `crawl_html(site)` 统一入口。
- 用 `@register("site_name")` 装饰器注册每个 HTML 站点的解析函数。
- 通过 `_get()` 统一处理 HTTP 请求、超时、异常和编码。
- 通过 `_make_article()` 统一生成文章结构。
- 通过 `_is_junk_title()` 集中过滤导航词、栏目名、品牌名、订阅入口等非新闻标题。

已实现的 HTML 站点解析函数：

| 函数 | 对应站点 | 关键逻辑 |
|------|----------|----------|
| `crawl_cnnpn()` | `cnnpn_domestic`、`cnnpn_international` | 解析核电信息网列表，日期优先来自列表项内标签 |
| `crawl_cpnn()` | `cpnn_energy`、`cpnn_tech` | 用 `resp.url` 拼接相对链接，从 URL 提取日期 |
| `crawl_sciencenet()` | `sciencenet` | 只抓正文列表相对链接，日期从父级 `<tr>` 文本提取 |
| `crawl_xinhua_tech()` | `xinhua_tech` | 只保留 `/tech/YYYYMMDD/hash/c.html` 文章页 |
| `crawl_netease_pv()` | `netease_pv` | 使用网易移动端静态页和 iPhone UA |
| `crawl_inen_solar()` | `inen_solar` | 解析 `X天前`、`X小时前`、`MM-DD` 等相对时间 |
| `crawl_renewablesnow()` | `renewablesnow` | 从图片 `alt` 提取标题，从父容器文本匹配英文日期 |
| `crawl_solarbe()` | `solarbe_tech` | 从 `/YYYYMM/DD/ID.html` URL 提取日期 |
| `crawl_tgs4c()` | `tgs4c` | 匹配 `slug-nidXXXX.html`，解析英文日期 |
| `crawl_china_nengyuan()` | `china_nengyuan` | 列表页无日期，保留为 `publish_time=None` |

扩展 HTML 站点时，需要在该文件新增解析函数并注册：

```python
@register("new_site_name")
def crawl_new_site(site: dict) -> list[dict]:
    ...
```

### `carbon_spider/utils/time_parser.py`

时间解析和日期过滤工具。

主要职责：

- `parse_time()` 将不同格式的日期字符串解析为 `datetime`。
- 支持 `YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY.MM.DD`、`YYYY年MM月DD日` 和 RSS 常见英文时间格式。
- `is_today()` 判断是否为当天。
- `is_yesterday()` 判断是否为昨日；当时间为空时返回 `True`，避免因缺日期漏抓。

注意事项：

- 当前日期比较按 UTC 日期执行。
- `publish_time=None` 的文章会通过昨日过滤，因此无日期站点可能混入历史文章。

### `carbon_spider/spiders/__init__.py`

Python 包初始化文件，使 `spiders` 目录可以作为模块导入。

### `carbon_spider/utils/__init__.py`

Python 包初始化文件，使 `utils` 目录可以作为模块导入。

主要依赖：

## 三、配置文件说明

### `carbon_spider/configs/sites.yaml`

全部数据源配置文件，是抓取范围的核心清单。

每个站点通常包含：

| 字段 | 作用 |
|------|------|
| `name` | 站点唯一名称，对应输出中的 `site` |
| `category` | 新闻分类 |
| `home_url` | 站点首页或列表页 |
| `rss_url` | RSS 地址，仅 RSS 站点需要 |
| `type` | 抓取类型，`rss` 或 `html` |

当前配置包括 26 个站点：

- 14 个 RSS 源，适合标准化抓取。
- 12 个 HTML 列表页，需要在 `html_spider.py` 中逐站解析。

新增站点规则：

- 新增 RSS 站点通常只需要修改 `sites.yaml`。
- 新增 HTML 站点需要同时修改 `sites.yaml` 和 `html_spider.py`。

### `.github/workflows/daily_scrape.yml`

GitHub Actions 定时抓取任务。

主要流程：

1. 每天 UTC 04:00 触发，即北京时间 12:00。
2. Checkout 仓库。
3. 安装 Python 3.10 和依赖。
4. 执行 `cd carbon_spider && python3 main.py --no-content`。
5. 将 `carbon_spider/outputs/daily_news_*.csv` 复制到 `daily-news/`。
6. 如果 CSV 有变化，则自动 commit 并 push。

该文件保证仓库可以无人值守地每日更新新闻归档。

### `.gitignore`

忽略本地运行和缓存文件。

当前主要忽略：

- Python 缓存：`__pycache__/`、`*.py[cod]`
- Codex 本地配置：`.codex`、`.codex/`
- 本地临时输出：`carbon_spider/outputs/`
- 测试缓存：`.pytest_cache/`

## 四、数据文件说明

### `daily-news/daily_news_YYYYMMDD.csv`

每日新闻归档文件，已提交到 Git 仓库。

当前仓库中已有从 `20260413` 到 `20260517` 的 35 个 CSV 文件。每个文件名中的日期表示新闻日期，而不是运行日期。

CSV 字段：

| 字段 | 说明 |
|------|------|
| `date` | 新闻日期，格式 `YYYYMMDD` |
| `site` | 数据源名称 |
| `category` | 分类 |
| `title` | 标题 |
| `url` | 原文链接 |
| `publish_time` | 发布时间 |
| `content` | 正文，默认空 |

这些文件是项目的主要产出，可直接用于检索、分析、汇总或后续数据处理。

本地临时输出目录，由 `main.py` 自动创建。

## 七、运行与数据流

完整数据流如下：

```text
GitHub Actions 或手动执行 main.py
    ↓
读取 carbon_spider/configs/sites.yaml
    ↓
按 type 分发
    ├── rss  → carbon_spider/spiders/rss_spider.py
    └── html → carbon_spider/spiders/html_spider.py
    ↓
carbon_spider/utils/time_parser.py 解析时间并过滤昨日新闻
    ↓
写入 carbon_spider/outputs/daily_news_YYYYMMDD.csv
    ↓
GitHub Actions 复制到 daily-news/
    ↓
自动 commit 并 push
```

## 八、维护注意事项

- 修改站点配置后，建议先运行 `python3 main.py --no-content --all-dates` 验证是否能抓到结果。
- 修改日期过滤逻辑时，要注意 `publish_time=None` 当前被设计为保留。
- 修改 HTML 解析函数时，要同步检查垃圾标题过滤是否会误删真实新闻。
- 对于被 WAF 或 Cloudflare 拦截的网站，单纯更换 UA 不一定有效，可能需要 Playwright，但会增加运行时间和依赖复杂度。
- `daily-news/` 是正式归档目录，`carbon_spider/outputs/` 只是本地临时输出目录。
